# BOSS 求职猎手产品说明

## 目标

当用户主动发起求职搜索时，帮助用户在 BOSS 上找到更适合的公司和岗位，并提升拿到面试的概率。这个 skill 会综合候选人画像、原始简历、岗位发现、匹配评分、简历优化建议、面试准备和可选的长期记忆。

## 主要用户

有简历和求职偏好，但不想手动翻大量 BOSS 岗位的求职者。用户希望 agent 能在获得授权后复用候选人画像，按需搜索、筛选岗位，并解释哪些公司最值得投入精力。

## 核心任务

- 把简历转成可搜索的候选人画像。
- 把偏好拆成硬性筛选条件和软性排序信号。
- 在用户同意后跨会话复用候选人状态、原始简历上下文和求职偏好。
- 不要求用户先提供 JD，也能主动在 BOSS 上搜索相关岗位。
- 按匹配度、面试概率和战略价值排序岗位。
- 解释简历缺口，并说明如何真实地强化特定岗位的表达。
- 为最有希望的岗位生成面试准备建议。
- 在用户同意后记录已过滤公司、反复出现的短板和长期提升点。

## 非目标

- 不自动投递岗位。
- 不自动和招聘者聊天。
- 不做定时、周期性或后台搜索。
- 不绕过登录、验证码、反爬或平台访问限制。
- 不承诺 offer 或保证面试结果。
- 不编造简历经历。
- 不在未获同意时静默持久化敏感简历或画像数据。

## 第一版范围

第一版只支持一次用户主动发起的搜索会话：

1. 用户提供简历和求职要求。
2. 如果已有用户批准的候选人状态，agent 可以复用。
3. 用户主动请求后，agent 通过可用浏览器访问 BOSS。
4. agent 检查有限数量的可见岗位。
5. agent 输出排序后的推荐、证据、简历建议、面试准备和建议记忆更新。
6. 用户同意后，agent 才持久化候选人画像、偏好、公司记忆和成长记录。

更丰富的搜索历史学习、公司记忆、投递和面试状态可以放到后续版本。自动监控和每日重复搜索仍然不在范围内，除非之后明确改变产品边界。
