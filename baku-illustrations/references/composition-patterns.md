# 构图模式与原创规则

## 基础结构类型

选择一种结构即可，不要混太多。

### Workflow 流程

适合：输入 -> 处理 -> 输出，内容生产，AI 工作流，自动化链路。

画法：左侧输入，中间 Baku 在工作台上处理，右侧输出。用 Amber 或 Work Blue 表达主流向。

### 证据链

适合：调试、复盘、判断依据、决策过程、申报口径。

画法：几个证据纸条从不同来源汇入 Baku 的桌面，最后形成一个明确结论。重点是“来源 -> 筛选 -> 结论”。

### 系统局部

适合：信息来源、过滤器、数据库、渲染器、agent 系统局部。

画法：只画 3-5 个核心模块。Baku 参与其中一个关键动作，例如修一根线、校验一个字段、把日志放进筛子。

### 前后对比

适合：混乱/有序，手动/自动，分散/收拢，焦虑/稳定。

画法：左侧凌乱但不要吵，右侧清楚可执行，中间一个克制箭头。Baku 做的是整理动作，不是夸张表演。

### 角色状态

适合：创作者状态、开发者焦虑、工具太多、从卡住到跑起来。

画法：2-4 个小状态，每个状态一个短标注。保持留白，不要变成漫画海报。

### 工作台隐喻

适合：内容工厂、信息仓库、脑内黑盒、自动日报、提示词整理、知识沉淀。

画法：一个大的桌面物件或低科技装置，少量输入，一个输出。要有记忆点，但不能喧宾夺主。

### 方法分层

适合：方法论框架、系统层级、能力栈、内容系统分层。

画法：一层层纸板、抽屉、便签或台阶，不要正式金字塔。Baku 在旁边搭建、标记或校验。

### 地图路线

适合：从想法到上线、用户路径、内容承接路径、学习路线。

画法：一条弯曲但清楚的路径，少量节点。Baku 拿笔标出下一步或把路牌扶正。

## 原创隐喻生成法

每次都从当前文章重新发明隐喻，不能照搬旧图。

### 三步

1. 把抽象概念换成一个物理动作：筛选、连接、折叠、压缩、修补、校验、归档、拆包、回流、对齐。
2. 把系统结构换成一个低科技工作台物件：便签墙、抽屉、线轴、台灯、筛子、剪刀、印章、秤、文件夹、卡片盒、旧机器、白板。
3. 让 Baku 承担动作：不是站旁边，而是在剪线、贴标签、调台灯、修链路、拿放大镜看日志、把纸条塞进筛子、把结论递出去。

### 可用物件池

- 电脑、终端窗口、笔记本、便签、文件夹、卡片盒、抽屉、台灯、咖啡杯、植物、线轴、白板、剪刀、印章、筛子、秤、旧机器、纸箱、路牌。
- 用时只选 1-3 个，不要堆满。

### Baku 动作池

- 读、圈、连、剪、贴、筛、称、修、拆、压、归档、递交、校验、标记、整理、照亮。
- 动作要服务核心意思，不要为了装饰而加动作。

## 反复刻规则

不要默认打开或复刻参考仓库案例。小黑和 Esther 的价值是迁移方法，不是构图库。

除非用户明确说“照这张 / 复刻这个构图 / 用这个案例改”，否则不要复用这些旧方向：

- 小黑纯白背景上的黑色怪物拉线、卡机器、切鱼、盖章。
- Esther/不二的浅蓝卷发、红蝴蝶结、黄裙、彩色蜡笔女孩主角。
- 大面积彩色蜡笔填充和深蓝粗描边。
- 把文章主题做成正式流程图或课程页。

同类主题也要换新隐喻。例如“证据链”不一定画链条，可以画 Baku 在台灯下把三张证据卡片叠成一个小结论盒；“自动化流程”不一定画箭头，可以画一根桌面线轴穿过三个夹子。
