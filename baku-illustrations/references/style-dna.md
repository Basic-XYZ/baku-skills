# 风格 DNA

## 一句话

暖纸、克制、手绘、工程工作台、中文批注、证据链清楚、个人 IP 参与核心动作。

像一个长期做工程、产品、AI 工具和知识整理的人，在桌面笔记上把复杂问题拆成一张能马上看懂的图。

## 必须

- 16:9 横版正文配图。
- 背景使用暖纸色或接近白底的干净纸面：参考 `#F7F4EA`，可以轻微纸张温度，但不要复古脏纸。
- 线条以 Ink 黑为主：`#161A22`，手绘、克制、略有抖动，不机械，不矢量，不厚重描边。
- 人物默认使用 2D 蜡笔 / 铅笔 / 水彩手绘质感，可带白色贴纸描边；不要默认生成 3D 角色。
- 大量留白：主体占画面约 40%-60%，至少 30% 空白。
- 少量中文手写批注：最多 5-8 处，每处尽量 2-8 个字。
- 一张图只讲一个核心动作、结构、状态或隐喻。
- Baku 必须是动作主体，参与拆解、连接、修补、筛选、记录或交付。
- 结构要自然表达，不要在图上写结构类型名称。

## 颜色

采用 Baku 品牌色，但用于插图时要更克制：

- Ink `#161A22`：主线稿、角色轮廓、核心物件、主要文字。
- Paper `#F7F4EA`：背景。
- Field `#E9E2D0`：纸条、便签、面板、低对比区域。
- Baku Yellow `#F6D21F`：角色背景记忆点、灯光、关键证据贴纸或少量强调。
- Signal Green `#18A058`：可执行、通过、结果稳定。
- Work Blue `#2563EB`：系统状态、链接、工具、AI 辅助层。
- Amber `#D97706`：主路径、时间线、风险提醒。
- Vermilion `#D64B3A`：错误、告警、少量重点。

比例：Ink/Paper/Field 70%，Baku Yellow 15%，Signal Green/Work Blue 10%，Amber/Vermilion 5%。红色只能点到为止。

## 文字

- 中文标注短、具体、像手写便签，不像课件标题。
- 可以出现少量英文技术词，但不要让画面变成英文 UI。
- 不要写大标题；尤其不要在左上角写“Workflow / 系统架构图 / 常见坑 / 路线图”。

## 绝对不要

- 不要商业 SaaS 插画。
- 不要 PPT 信息图。
- 不要正式流程图。
- 不要课程课件。
- 不要可爱卡通海报。
- 不要儿童插画。
- 不要复杂架构图。
- 不要蓝紫渐变、霓虹、玻璃拟态、AI 光效。
- 不要圆胖 3D 娃娃脸、玩具感头像或默认 3D mascot。
- 不要真实 App 截图或 stock photo 背景。
- 不要把每个节点都解释清楚。
- 不要照搬小黑的纯白黑线怪诞草图，也不要照搬不二的彩色蜡笔女孩设定。

## 审美方向

要实用、克制、可信、有个人识别度。

可以有一点轻微荒诞，但荒诞必须服务理解；不要为了怪而怪。
