# Baku IP

## 角色定义

Baku 是这个 skill 的固定视觉 IP：一个程序员型个人形象，代表“把复杂问题拆成可执行路径的人”。

默认每张图都要出现 Baku。Baku 不是贴纸，不是普通头像，不是可爱吉祥物，而是正在参与系统运转、判断、连接、修补或交付的工作台角色。

## 外形

- 东亚男性成人气质，30 岁左右。
- 短到中等长度的深黑色蓬松碎发，有自然层次和手绘铅笔质感；额前可以有碎刘海或轻微偏分，但不要中分、不要整齐锅盖头、不要贴头皮。
- 圆形或近圆形细框眼镜，镜框要轻，不要厚重黑框。
- 黑色 T 恤，必要时可以加简单外套，但不要 logo。
- 脸型偏清瘦、自然椭圆或略窄，颈部和肩膀比例成人化；不要圆胖脸、鼓脸颊、娃娃头或玩具感。
- 表情平静、认真、友好，可以有轻微腮红和安静的少年感，但不要夸张卖萌。
- 常出现在电脑、笔记本、便签、代码符号、植物、咖啡杯、桌灯旁。
- 黄色背景或黄色便签是主要记忆点，但不要求每张都有大块黄色。

`assets/avatar.jpg` 是角色校准素材。使用它理解发型、眼镜、黑 T 和工作台气质即可，不要把它当作必须逐像素复刻的参考图。

## 角色风格优先级

默认优先 2D 手绘角色，不默认走 3D：

1. 蜡笔 / 铅笔质感贴纸风：深蓝线条、黑色粗糙填色、白色贴纸描边、黄色背景或黄色点缀，适合正文插图和工作台图。
2. 温暖水彩 / 手绘动画风：清瘦脸型、自然光、轻量植物或桌面背景，适合更安静的解释图。
3. 现代简化 2D 漫画风：清楚轮廓、成人比例、简单手势，适合流程说明。

除非用户明确要求，不要使用 3D。必须用 3D 时，也要保持清瘦成人比例、较窄脸型、较小脸颊、克制眼睛和非玩具质感。

## 性格

- 有工程判断，不炫技。
- 可信、克制、细致。
- 会把抽象问题变成证据链、流程、检查项和最小行动。
- 像真实 builder，不像营销博主、游戏角色或儿童角色。
- 可以温和幽默，但不要表情包化。

## 常见职责

让 Baku 承担核心动作：

- 把混乱纸条归档成证据链。
- 用线连接输入、判断、输出。
- 修补断掉的自动化链路。
- 从日志/文档/数据库里筛选有效信号。
- 给风险贴标签。
- 在工作台上压缩一堆想法，输出一张清楚卡片。
- 把复杂系统拆成 3-5 个可读部件。
- 把“还不确定”放到待验证盒子里。
- 把文章段落变成可执行路径。

## 禁止

- 不要把 Baku 画成小黑、不二或其他参考仓库角色。
- 不要画成 chibi、玩具、儿童卡通、游戏 mascot。
- 不要中分发型、圆胖脸、鼓脸颊、球形头、过大的娃娃眼。
- 不要默认做 3D 软萌角色；3D 只在用户明确要求时使用，并且要更瘦、更成人、更克制。
- 不要过度科技感、赛博朋克、霓虹蓝紫。
- 不要给复杂服装、夸张表情、恶魔角、盔甲或奇幻设定。
- 不要让 Baku 只是坐在角落看图表。
- 不要让角色抢走结构表达。

## 判断标准

如果去掉 Baku，图的核心隐喻还能完全成立，说明 Baku 太装饰了；要重写提示词，让他成为动作主体。
