# 如何把这个 skill 复刻成个人风格

当用户说“我也想做一个自己的插图 skill”“怎么做我自己的版本”“这个风格挺好看，怎么迁移”时读取本文件。

## 核心原则

不要只改角色名。一个个人风格插图 skill 至少要同时迁移六件事：

1. 角色 IP：是谁、长什么样、性格是什么、在图里承担什么动作。
2. 视觉 DNA：背景、线条、颜色比例、质感、留白、文字密度。
3. 构图语法：常用结构类型、隐喻生成方法、物件池、动作池。
4. Prompt 模板：每次生成时必须写进去的正向要求和负向约束。
5. QA 标准：什么算合格、什么必须重生成、怎么迭代。
6. 公开说明：README 或说明文档要告诉使用者这个 skill 解决什么问题，怎么安装，怎么继续改成自己的。

迁移个人风格时有一个明确经验：不能只替换名字，必须同时替换角色、色彩、线条质感、构图语法、prompt 模板和 QA 标准。迁移时要继续把 references 和 QA 一起改干净，避免旧角色规则残留。

## 最小制作流程

### 1. 收集 6-12 张风格证据

来源可以是：

- 自己已有头像、插图、封面、图文卡片。
- 你喜欢但不能照抄的风格参考。
- 你明确不喜欢的反例。
- 真实输出里最稳定的 3-5 张。

把它们分成三类：

- `identity`：角色长相和个人识别点。
- `style`：线条、颜色、背景、质感。
- `composition`：图怎么组织信息。

### 2. 写一句定位

格式：

```text
为 {内容场景} 生成 {比例/用途} 插图，把 {内容类型} 变成 {视觉表达}，默认使用 {角色 IP} 和 {风格关键词}。
```

例子：

```text
为中文工程文章生成 16:9 正文配图，把流程、判断和证据链变成克制的个人工作台手绘解释图，默认使用 Baku 个人 IP 和暖纸手绘风格。
```

### 3. 重写角色 IP

至少写清楚：

- 年龄/气质/身份。
- 发型、眼镜、服装、标志性道具。
- 表情和动作边界。
- 角色必须在图里做什么。
- 禁止像什么。

### 4. 重写风格 DNA

至少写清楚：

- 背景色和质感。
- 线条风格。
- 主色和强调色比例。
- 中文标注数量。
- 留白比例。
- 绝对不要项。

不要说“高级、好看、精致”这种空词，要能变成 prompt 约束。

### 5. 重写 prompt 模板

模板必须包含：

- 画幅和用途。
- 角色 IP 必须参与核心动作。
- 风格 DNA。
- 当前主题、结构、核心意思、构图、元素、标注词。
- 颜色用法。
- 负向约束。

### 6. 重写 QA

QA 要像验收表，不像审美散文。写出：

- 必过项。
- 失败信号。
- 迭代方法。
- 交付判断。

### 7. 放入 skill 结构

推荐结构：

```text
your-illustrations/
├── SKILL.md
├── README.md
├── agents/
│   └── openai.yaml
├── assets/
│   ├── avatar.jpg
│   └── examples/
└── references/
    ├── style-dna.md
    ├── your-ip.md
    ├── composition-patterns.md
    ├── prompt-template.md
    ├── qa-checklist.md
    └── personal-style-skill-guide.md
```

示例图可以先不放；但如果要放，必须是你自己的风格样张，且在 `SKILL.md` 里明确“只作校准，不要照抄构图”。

## 迁移检查

- `SKILL.md` frontmatter 的 `description` 是否写清楚触发场景。
- 所有旧角色名是否清干净。
- 所有颜色、线条、质感是否属于新风格。
- prompt 模板里是否还残留旧角色动作。
- QA 是否能阻止“只是换皮”的输出。
- README 是否告诉别人如何安装和如何复刻。

如果这些都过了，才算从参考 skill 迁移成了个人风格 skill。
