# Components

优先从这些组件类型中选择，再按具体内容写 HTML/CSS。

## Brand Header

- 左侧品牌名 `Baku` 或 `Basic-XYZ`。
- 右侧使用简短导航或状态 chips。
- 不做大面积玻璃拟态，不使用模糊背景。

## Evidence Card

用于承载结论、证据、影响和下一步。卡片半径不超过 `8px`，边框优先用 `#D8D0BD`，阴影要弱。

## Command Button

按钮使用明确动作文案，配 icon 时优先使用现有图标库。主按钮使用 Work Blue 或 Signal Green，危险动作使用 Vermilion。

## Metric Strip

适合展示数字、状态、进度。数字用较大字号，标签小而清楚，不使用花哨渐变数字。

## Quote Block

引用块必须有背景、边框、来源或上下文说明。不要使用默认 `blockquote` 或单纯 `border-left`。

## Table

表格用于比较和决策。表头需要浅底色，单元格留足 padding，移动端可横向滚动。

## Avatar Badge

头像默认可用 `assets/avatar.jpg`，新页面优先从 `assets/avatars/` 按场景选择。建议圆形或轻微圆角；旁边可以放身份标签：`builder`, `operator`, `design system`。如果页面是暗色或复古未来风，优先选择 dark、marker 或 line-art 系列；如果是暖白文档或亲和型页面，优先选择 light、anime-warm 或 yellow-sticker 系列。

## Timeline

每项包含时间/阶段、事件、证据和状态。适合工作流、项目进度、排障过程。
