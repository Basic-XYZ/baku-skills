# Scene: Tutorial / Explainer

教程页要像一份可执行的工程笔记，而不是博客模板。

## 结构

1. 首屏：主题、适用对象、完成后能得到什么。
2. 前置条件：环境、输入材料、风险提醒。
3. 分步骤说明：每步包含目的、动作、验证方式。
4. 示例输出：命令、截图位、表格或结果片段。
5. 收尾：常见错误、下一步、可复用模板。

## 视觉

- 使用 Document Split、Evidence Stack、Command Strip。
- 代码块和命令块要可读，避免整页终端风。
- 重点提示使用 Amber，不要满页警告色。

## 文案

- 直接说“做什么”和“怎么验证”。
- 少用形容词，多用可观察结果。
