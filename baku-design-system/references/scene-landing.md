# Scene: Landing / Event

Landing page 应该第一眼给出 Baku 的具体产品、服务、活动或作品，不做空泛大标题。

## 结构

1. Hero：品牌/产品/活动名，配真实对象、头像或工作台视觉。
2. 价值：用 3-4 个具体场景说明解决什么问题。
3. 证据：案例、产物截图、工作流、指标或用户反馈。
4. 行动：报名、联系、安装、查看文档。
5. 补充：FAQ、边界、适合/不适合谁。

## 视觉

- Hero 不放卡片里；可以使用全宽工作台或头像签名布局。
- 让下一段内容在首屏底部露出一点。
- 避免 split hero 里左文右图都像模板。
