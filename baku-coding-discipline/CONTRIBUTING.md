# 贡献指南

保持改动小而有证据。这个仓库本身也受它描述的工程纪律约束。

## 贡献规则

- 只改当前问题或改进所必需的文件。
- 保持这个 skill 作为工作流路由器的定位，不把它写成大型手册。
- 长清单或模式细节放到 `references/`。
- 保持 `SKILL.md` 足够简洁，让 Agent 能快速加载。
- 提 PR 前运行 `python3 scripts/validate.py`。

## 合适的改动

- 澄清含糊的路由规则。
- 补充缺失的验证标准。
- 去掉入口文件和 reference 之间的重复。
- 改进安装或校验说明。

## 避免

- 为罕见场景添加推测性工作流。
- 塞入无关的工程观点。
- 明明小改就能解决，却重写整个 skill。
- 为简单静态检查增加依赖。
